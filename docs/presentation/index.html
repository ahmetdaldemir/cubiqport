CubiqPort

Sunucu orkestrasyon platformu — tek panelden sunucular, domainler, DNS, SSL ve deployment

Node.js Fastify Next.js 14 PostgreSQL Redis Docker
1

Problem

Birden fazla sunucu ve domain yönetmek manuel, hataya açık ve zaman alıcı.

  • SSH ile her sunucuya ayrı bağlanıp nginx, SSL, DNS ayarları yapmak
  • Deploy için tekrarlayan script’ler ve farklı ortamlar
  • Metrik ve sağlık takibinin merkezi olmaması
  • Güvenlik: SSH anahtarları ve sırların güvenli saklanması

Hedef: Tüm sunucu ve domain yaşam döngüsünü tek bir kontrol panelinden yönetmek.

2

Çözüm: CubiqPort

Production-ready sunucu orkestrasyon platformu.

Yönetim paneli (Web)

Sunucu ekleme, domain oluşturma, DNS, SSL, deployment tetikleme ve canlı izleme.

API

Auth, sunucu/domain/DNS/deployment yönetimi, metrik toplama. REST + Swagger.

Agent

Her yönetilen sunucuda hafif bir HTTP agent: deployment çalıştırma, nginx + SSL, metrik gönderme.

Domain oluşturulunca otomatik: DNS (Cloudflare) → nginx config → SSL (Let's Encrypt).

Domain oluşturma akışı

API’ye POST /domains → Cloudflare’da A kaydı → Agent’a nginx config yazdırma → nginx reload → domain active.

6

Güvenlik

  • SSH anahtarları veritabanında AES-256-GCM ile şifreli saklanır
  • Tüm API rotaları JWT ile korunur; agent X-Agent-Secret (HMAC) ile doğrulanır
  • Şifreler bcrypt (12 round) ile hash’lenir
  • Rate limiting: 200 istek/dakika/IP
  • Helmet güvenlik başlıkları (API ve agent)

Production deploy: deploy.sh ile SSH/rsync → sunucuda PM2 + Nginx.

7

Deployment

./deploy.sh [api|web|all] — SSH ile sunucuya bağlanır, rsync ile kod gönderir, build ve PM2 restart yapar.

# Sunucu yapısı (örnek)
/var/www/html/
├── .env, package.json, ecosystem.config.cjs (PM2)
├── packages/shared/dist
├── apps/api/dist    → PM2: cubiqport-api
└── apps/web/.next/standalone  → PM2: cubiqport-web

Nginx reverse proxy; Redis isteğe bağlı (kuyruk + canlı metrik önbelleği). Worker: cubiqport-worker (Redis varsa analiz kuyruğunu işler).
